# Daily Rate Brief Source / 每日运价简报原文模板（业务员视角）

> 这是「每日运价简报内部版」（发给业务员看的）的**结构模板**。
> 每天 08:30 cron 必须按这个**结构**重新填入今天事实源的真数据。
> 数据可以变（卖价、CLS、船名、有效期等），**结构 / 块划分 / 字段顺序不要变**。

## 结构规则（必须遵守）

1. **按船司 + 区域分块**，不要按目的港 (POD) 分块。例如 "PIL 红海"、"KMTC 蛇口出红海中转船"、"深圳 CUL 红海"、"TSL 印巴"、"SJJ 印巴" 各自一块。
2. **块标题**：船司缩写 + 区域（"红海" / "印巴" / "美西" / "欧地" 等）。如果是中转或特殊属性，加在标题里（"中转船" / "直航 小柜免重"）。
3. **每行格式**：`POL-POD USD<20GP>/<40GP>` 后接可选备注。POL 用港口缩写（SK / NS / DCB / YT 等），不要写全称中文（不要 "SK蛇口"，直接 "SK"）。POD 保留英文 + 港名缩写。
4. **CLS / 船期行**：每块底部一行或多行，写明截关日 + 船名 + 航次号 + 操作变更（如"原船 X omit SHK，Y 替代收货"）。
5. **附加费 / 免柜 / 限重 / 跳港 / 中转天数**：直接写在所属行末尾或块末尾的备注行。
6. **单询占位**：HODEIDAH、AQABA 等如果某船司没具体报价但接受询价，写 "实单单询 JIB"（或对应中转港）形式，**不要拦截**。
7. **块之间用 `-------------------------` 分隔**（长度可不严格）。
8. 块内不要混入成本价、底价、利润、加价策略 — 这些只在内部 review 段，不进这份给业务员看的简报正文。

## 字段映射（从事实源到简报的提取规则）

| 简报里要出现 | 从事实源哪个字段抽 |
|-------------|-------------------|
| 船司块标题 | 「船司/服务」+「航线/区域」 |
| POL-POD 行 | 「起运港POL」(取缩写部分) + 「目的港POD」 |
| USD 20GP/40GP | 「卖价」中提取数字（剥离卖价附带的旧船期/备注文本）|
| 中转天数 | 「航程」字段，或「原始备注」里 "X 中转约 N 天" 模式 |
| 免柜期 | 「免柜期」字段 |
| 附加费 | 「附加费/本地费」字段（保留 EIS / CIC 等具名） |
| CLS / 船期 | 「截关/开船」字段（统一为 5 月 / 当日有效期）+「原始备注」里船名/航次号 |
| 操作变更 / 跳港 | 「限制条件」字段 + 「原始备注」里 omit / 替代收货 / 跳港 等关键句 |
| 单询占位 | 当某 POD 在该船司有港名但卖价为空时，写 "实单单询 JIB"（或对应中转港）|

---

## 不允许出现在内部简报里的内容

- **成本价、底价、采购价、拿价、cost、net、内部利润、加价策略** — 这些进**内部 review 单独段**，不进给业务员看的简报正文。
- **OneNote 段段抄录的乱文本** — 必须拆字段再呈现，不能整段贴。
- **字段错位 / 数值异常**（如 "20GP=USD10"）— 进风险拦截段，不进正文。

---

## 顶部统计（最新要求：plain txt / no style）

主结构（船司分块）保留之外，简报顶部最多加一行紧凑统计，例如：
`[运价表 77 行 / 运价信息 9 段 / SCFI 1954.21 +2.24%]`

不要在发给业务员的每日简报正文里加入：
- SCFI 表格或市场分析大段
- 总览章节
- Top 5 章节
- 风险拦截 / 待业务确认章节
- Markdown 标题、表格、code fence、加粗、emoji、PDF/HTML 样式

字段错位、过期 CLS、数值异常等只写入本地 `run-status.md` 和企微备注/审核依据，不作为 Telegram `.txt` 简报正文。主结构必须仍然是船司+区域分块。
